# Common Rules

Общие правила для подготовки и последующей автоматизации независимо от раздела.

## Основные принципы
- Не придумывать предметную логику при нехватке данных.
- Не подменять подтверждённые правила предположениями.
- Любое противоречие фиксировать явно.

## Классификация утверждений
Каждый новый тезис должен иметь один из статусов:
- `CONFIRMED` (подтверждено пользователем);
- `OBSERVED` (наблюдается в примерах/заметках);
- `HYPOTHESIS` (рабочее предположение);
- `OPEN_QUESTION` (требует решения).

## Границы общего ядра
- Общие механики (нормализация, валидации, комплектность) хранятся в общих правилах.
- Различия разделов (ВК/OV1/OV2) хранятся в `sections/<SECTION>/`.
- Нельзя вшивать логику конкретного раздела в общий слой.

## Про изменения
- Любое изменение логики сопровождается обновлением документов в `docs/` и соответствующем `sections/`.
- Старые версии алгоритма не считаются неизменными.

## Качество входных данных
- Приоритет у реальных актуальных входов текущего проекта.
- Исторические заметки и прошлые примеры используются как материал для гипотез и наблюдений.

## Правило эскалации неопределённостей
- Если нет данных для однозначного решения, вопрос переносится в `docs/open-questions.md`.
- Запрещено «молчаливо выбрать» одну из конфликтующих версий.
