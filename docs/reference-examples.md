# Reference Examples Analysis

Источник: `references/example-projects/vk-similar-project`.

## Что подтверждается примером с высокой уверенностью
1. Структура по секциям
- Есть отдельные каталоги вида `ВК - 13.1`, `ВК - 13.2`, `ВК - 13.3`.

2. Базовые папки внутри секции
- `Акты/`, `Сертификаты/`, `Схемы/`.
- В части секций также присутствует `Комплекты/` и папки печати.

3. Типы файлов в рабочем наборе
- Word (`.docx`, местами `.doc`), PDF, Excel (`.xlsx`), DWG.
- Много сертификатов хранится как PDF/JPG.

4. Наблюдаемая структура актов
- Акты разложены по системам: например `В0`, `В2`, `В21`, `К1.1`, `К1.2`, `К2` и т.д.
- Есть отдельные файлы с пометками типа акта (`Гидравлика`, `Промывка`, `Дезинфекция`, `Пролив`, `Испытания`, `АОУСИТО`).
- Есть файлы `прил` (приложения).

## Что подтверждается со средней уверенностью
- В именовании часто встречается паттерн: `№<номер>-<система>_<секция> [тип].docx`.
- В корне секции обычно лежат шаблоны актов и рабочая Excel-книга актов.
- Итоговый комплект может поставляться в zip-архиве.

## Что пока остаётся предположением
- Финальные обязательные правила именования для всех проектов.
- Жёсткий список обязательных типов актов для любой секции.
- Единый состав финального архива для всех заказчиков.

## Количественные наблюдения (по текущему примеру)
- Всего файлов в примере: 535.
- Файлы сертификатов по секциям: по 62 в `13.1`, `13.2`, `13.3`.
- PDF-схемы: 31 (`13.1`), 21 (`13.2`), 25 (`13.3`).
- Наблюдаемые ключевые метки в названиях Word-файлов: `АОУСИТО`, `прил`, `Гидравлика`, `Промывка`, `Дезинфекция`, `Пролив`, `Испытания`.

## Ограничения анализа
- Пример полезен как референс, но не является утверждённой спецификацией.
- Часть файлов служебная (например, временные файлы офисных приложений) и не должна попадать в правила процесса.
