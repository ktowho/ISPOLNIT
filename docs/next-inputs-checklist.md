# Next Inputs Checklist

Цель: собрать минимально достаточный реальный набор файлов для перехода от подготовки базы к анализу реального процесса.

## Обязательный минимум
1. 1-2 реальных Word-реестра по ВК
- Нужны для проверки структуры полей и правил извлечения.

2. Пример журнала работ (Excel/Word/PDF)
- Нужен для правил подстановки и сверки дат.

3. Пример таблицы материалов/замен оборудования
- Нужен для валидации связки «материал -> сертификат».

4. Набор Word-шаблонов актов
- Минимум: основной акт + 1-2 нестандартных типа.

5. Пример сертификатов на материалы
- Нужны для проверки правил комплектности и приложения 6+.

6. Один реальный итоговый комплект по ВК
- Нужен для верификации структуры выдачи и именования.

## Желательно добавить сразу
1. Пример с замечаниями после первой проверки
- Чтобы формализовать типичный цикл доработок.

2. Пример, где есть спорные/неполные данные
- Чтобы заранее зафиксировать правила обработки неопределённостей.

3. Пример из другой секции/объекта
- Чтобы проверить переносимость правил и не переобучиться на один кейс.

## Критерий готовности следующего этапа
- Для каждого пункта есть хотя бы один реальный файл.
- По каждому файлу зафиксирована роль в процессе (вход/референс/выход).
- Невязки и пропуски занесены в `docs/open-questions.md`.
