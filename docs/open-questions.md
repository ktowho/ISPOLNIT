# Open Questions

Все вопросы ниже требуют явного решения и не должны закрываться молчаливыми допущениями.

## Закрыто (подтверждено пользователем, 2026-03-26)
1. При конфликте источников важнее примечания в реестре.
2. Журнал используется для сверки дат; отклонения 1-2 дня допустимы, если нет явных нестыковок.
3. Дата окончания работ не должна быть позже закрытия.
4. Форматы дат в документах могут отличаться по месту; ориентир — формат конкретного поля по примерам.
5. Именование акта: `№<номер>-<система>_<секция>.docx` (обязательный символ `№` в начале).
6. Для приложений/испытаний/промывки и т.п. добавляется соответствующая приписка в конце имени файла.
7. Состав обязательных актов определяется реестром; от реестра не отклоняться, кроме исправления явной грамматической опечатки.
8. Сертификаты обрабатываются по реестру; дополнительные сертификаты в папке не являются входом для формирования.
9. Правило приложения жёсткое: если сертификатов 6 и более — отдельное приложение; если 5 и меньше — реквизиты в акте.
10. Допустим единый рабочий табличный контур, но с разбиением по секциям сверху вниз в соответствии с реестрами.
11. В текущем этапе помощь по рисованию схем не требуется.
12. Выбор стека для будущей реализации: Python.
13. Содержимое исходных актов не изменяем; допустимо только переименование с добавлением `№` в начало, если символ отсутствует.
14. Эталон для названия объекта, ФИО подписантов, номеров приказов и реквизитов организаций — существующие акты `Промывка` и `Дезинфекция`; АОУСИТО для этого не используется.
15. Основная рабочая таблица ведётся отдельно от журнала работ: журнал остаётся самостоятельным источником для будущего заполнения дат. Рабочую таблицу хранить вне `inputs/` как новый производный артефакт (подтверждено пользователем, 2026-03-27).
