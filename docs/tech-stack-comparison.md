# Tech Stack Comparison: Python vs Node.js

Сравнение сделано для задач будущей автоматизации ИД: Word/Excel, шаблоны, нормализация данных, проверка комплектности.

## Критерии
1. Надёжность работы с Word/Excel в production-процессах.
2. Удобство нормализации и валидации табличных данных.
3. Контроль качества выходных документов.
4. Предсказуемость сопровождения.

## Сравнение
1. Работа с Excel
- Python: сильная экосистема (`openpyxl`, `pandas`), удобная обработка таблиц и проверок.
- Node.js: есть рабочие библиотеки (`exceljs`, `xlsx`), но обычно меньше удобства для сложной аналитики/валидаций.

2. Работа с Word-шаблонами
- Python: стабильный набор библиотек для шаблонного заполнения и пакетной обработки.
- Node.js: есть `docxtemplater` и аналоги, но для сложных кейсов чаще больше ручной доработки.

3. Нормализация и проверка комплектности
- Python: сильнее для data-centric задач и построения проверок качества.
- Node.js: возможно, но обычно с большим количеством вспомогательного кода на тех же сценариях.

4. Надёжность и контроль регрессий
- Python: проще реализовать пайплайн «извлечение -> нормализация -> валидация -> генерация» с повторяемыми проверками.
- Node.js: хорош для интеграций и сервисного слоя, но не даёт заметного преимущества для документно-табличного ядра.

## Практическая рекомендация
Выбирать `Python` как основной стек для ядра автоматизации документов.

Почему:
- лучше подходит для качества и надёжности в задачах Word/Excel + нормализация + валидации;
- быстрее приводит к предсказуемому результату при сложных табличных и проверочных сценариях;
- снижает риск ручных пост-правок при масштабировании процесса.

## Когда Node.js может быть полезен
- внешний интерфейс, интеграционные API, оркестрация вокруг Python-ядра.
- но документное ядро целесообразно держать в Python.
