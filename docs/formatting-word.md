# Word Formatting Rules (Draft)

Документ фиксирует правила подготовки Word-выходов в будущем автоматизированном контуре.

## Подтверждённые требования
- Формирование актов по Word-шаблонам.
- Один акт должен формироваться в отдельный Word-файл.
- Если на акт 6 и более сертификатов, должно формироваться приложение.
- Если сертификатов 5 и меньше, реквизиты сертификатов указываются в самом акте.
- Состав актов формируется строго по реестру.

## Практические правила подготовки шаблонов (`HYPOTHESIS`)
- Держать стабильные шаблоны по типам актов.
- Разделять общий шаблон и шаблоны нестандартных актов.
- Избегать ручных пост-правок, которые нельзя воспроизвести автоматически.

## Поля и форматирование (`HYPOTHESIS`)
- Нумерацию акта, систему и тип акта держать в согласованном месте документа.
- Для многострочных полей (материалы, сертификаты) использовать предсказуемую табличную структуру.

## Даты (`CONFIRMED`)
- Даты в разных местах документа могут иметь разный формат.
- Формат даты выбирать по примерам для конкретного места (поля), а не унифицировать насильно по всему комплекту.
- При сверке с журналом допустимы отклонения 1-2 дня, если нет явной логической нестыковки.
- Если для конкретного поля допустимы оба варианта (`01.01.2025 г.` и `01.01.2025г.`), в текущем проекте использовать вариант без пробела: `01.01.2025г.`.

## Именование файлов (`CONFIRMED`)
- Базовый шаблон: `№<номер>-<система>_<секция>.docx`.
- Символ `№` в начале обязателен.
- Для приложений/испытаний/промывки и т.п. добавляется приписка в конце имени файла.

## Риски из наблюдений
- Ручная табуляция и ручная проверка вставок занимают заметное время.
- Нужны чёткие требования к шаблонам, чтобы исключить постоянные локальные правки.
