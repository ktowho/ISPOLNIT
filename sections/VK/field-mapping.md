# VK Field Mapping (Draft)

Черновая карта полей между входами и внутренней структурой для ВК.

## Важно
- Это не финальная таблица колонок.
- До получения реальных входов текущего проекта названия полей считаются рабочими гипотезами.
- При конфликте данных приоритет у примечаний реестра (`CONFIRMED`).

## 1) Реестр актов (Word) -> Act
1. `Номер акта` -> `act.act_number` (`HYPOTHESIS`)
2. `Система` -> `act.system_code` (`HYPOTHESIS`)
3. `Тип акта` -> `act.act_type` (`HYPOTHESIS`)
4. `Дата завершения работ` -> `act.date_work_end` (`HYPOTHESIS`)
5. `Дата подписания` -> `act.date_sign` (`HYPOTHESIS`)
6. `Листы ИС/схемы` -> `act.schemes[]` (`HYPOTHESIS`)
7. `Материалы` -> `act.materials[]` (`HYPOTHESIS`)
8. `Сертификаты` -> `act.certificates[]` (`HYPOTHESIS`)

## 2) Журнал работ -> WorkLogEntry
1. `Вид работ` -> `work_log.work_type` (`HYPOTHESIS`)
2. `Дата начала` -> `work_log.date_start` (`HYPOTHESIS`)
3. `Дата окончания` -> `work_log.date_end` (`HYPOTHESIS`)
4. `Привязка к системе/участку` -> `work_log.scope_ref` (`HYPOTHESIS`)

## 3) Реестр сертификатов/файлы сертификатов -> Certificate
1. `Наименование материала/оборудования` -> `certificate.material_links[]` (`HYPOTHESIS`)
2. `Номер/идентификатор сертификата` -> `certificate.certificate_id` (`HYPOTHESIS`)
3. `Файл сертификата` -> `certificate.certificate_file` (`CONFIRMED` как сущность файла)

## 4) Бизнес-правила связки (`CONFIRMED`/`OBSERVED`)
1. Даты актов сверяются с журналом работ.
2. Допустимы отклонения дат 1-2 дня, если нет явных нестыковок.
3. Дата окончания работ не должна быть позже закрытия.
4. Материалы должны иметь связанный сертификат по реестру.
5. Если сертификатов на акт 6 и более, формируется приложение.
6. Если сертификатов 5 и меньше, реквизиты указываются в акте.
7. Выход формируется как отдельный Word-файл на каждый акт.
8. Именование файла: `№<номер>-<система>_<секция>.docx` плюс приписка типа при необходимости.

## 5) Что нужно подтвердить входами
1. Точные названия колонок в реестрах.
2. Правила обработки пропущенных/дублирующихся строк.
3. Формальные правила связки «материал -> сертификат», если названия не совпадают дословно.
